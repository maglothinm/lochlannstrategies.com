# Human Review Notes

## Source organization

The site is intentionally written for direct review rather than generated by a framework.

- Each page uses semantic landmarks: `header`, `nav`, `main`, `section`, `article`, and `footer`.
- Major page sections are separated with visible comment banners.
- Class names describe purpose, not appearance. Examples include `decision-card`, `credential-rail`, `engagement-card`, and `experience-timeline`.
- Repeated visual values are centralized in the design-token section at the top of `assets/site.css`.
- No inline styles are used.
- No minified files, utility-class chains, template languages, or generated component wrappers are required.

## Visual transition strategy

The site deliberately avoids alternating full navy and bright-white sections.

The primary surfaces are:

- `surface--mist` — muted blue-gray
- `surface--stone` — warm neutral
- `surface--paper` — warm ivory
- `surface--blue` — slightly stronger muted blue-gray

The home and interior heroes fade into the next light surface rather than ending on a hard color boundary. Dark treatments are primarily contained within the hero, CTA panels, contact panel, and footer. The footer begins with a slate transition before moving into deep ink.

## Common edits

### Colors, type, spacing, radii, and shadows

Edit the variables under `1. DESIGN TOKENS` in `assets/site.css`.

### Shared header and footer

The header and footer markup is repeated in each HTML file so the site remains deployable without a template engine. Update each page when changing shared navigation or footer content.

### Motion

Reveal motion is controlled in `16. PROGRESSIVE ENHANCEMENT`. The site respects `prefers-reduced-motion`. Removing `assets/site.js` leaves the content and navigation usable.

### About-page portrait

The current About page uses `assets/founder-perspective.svg`, an abstract operating-perspective illustration. It can later be replaced by a professional portrait without changing the surrounding content structure. Preserve meaningful `alt` text when doing so.

## Review checklist

Before deployment, verify:

1. Page titles and descriptions remain accurate.
2. Navigation labels and order match across pages.
3. Each page contains one `h1`.
4. New images include useful alternative text unless decorative.
5. New colors meet contrast requirements.
6. Internal links and section anchors resolve.
7. Desktop and mobile layouts are reviewed.
8. Keyboard navigation and focus indicators remain visible.
9. The site remains understandable with JavaScript disabled.
10. `sitemap.xml` reflects any added or removed pages.
